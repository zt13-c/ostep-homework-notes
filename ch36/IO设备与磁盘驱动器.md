# I/O 设备与磁盘驱动器笔记

## 1. 标准设备模型

操作系统与设备交互时，通常不会直接了解设备内部如何工作，而是通过设备暴露出来的接口进行控制。

一个标准设备可以分成两部分：

```text
设备 = 对外接口 + 内部结构
```

### 1.1 对外接口

设备对操作系统暴露一些寄存器，例如：

```text
状态寄存器 STATUS
命令寄存器 COMMAND
数据寄存器 DATA
```

操作系统通过读写这些寄存器控制设备。

| 寄存器 | 作用 |
|---|---|
| STATUS | 查看设备当前状态，例如是否忙、是否完成、是否出错 |
| COMMAND | 向设备发送命令，例如读、写、重置 |
| DATA | 用于传输数据 |

### 1.2 内部结构

设备内部可能有自己的：

```text
微处理器
内存
专用控制芯片
固件 firmware
```

复杂设备如 SSD、网卡、RAID 控制器内部也可能有自己的处理逻辑。

操作系统通常不关心设备内部怎么实现，只需要知道如何通过接口控制它。

---

## 2. 标准设备协议

一个简单的设备交互协议如下：

```c
while (STATUS == BUSY)
    ;   // 等待设备空闲

Write data to DATA register

Write command to COMMAND register

while (STATUS == BUSY)
    ;   // 等待设备完成请求
```

### 2.1 第一步：等待设备空闲

操作系统让 CPU 不断读取状态寄存器：

```c
while (STATUS == BUSY)
    ;
```

这叫做轮询，polling。

本质是：

```text
CPU 主动反复检查设备状态。
```

### 2.2 第二步：写入数据

如果是写设备，操作系统需要把数据写入设备的数据寄存器。

如果由 CPU 亲自搬运数据，这种方式叫：

```text
Programmed I/O, PIO
```

也就是编程 I/O。

### 2.3 第三步：写入命令

操作系统把命令写入命令寄存器，告诉设备开始执行。

例如：

```text
READ
WRITE
RESET
```

### 2.4 第四步：等待设备完成

设备执行期间，CPU 再次轮询状态寄存器，直到设备完成。

---

## 3. 轮询、中断、PIO、DMA 的区别

这几个概念容易混淆，需要分开理解。

### 3.1 PIO 和 DMA 解决的是“谁搬数据”

| 方式 | 谁搬数据 |
|---|---|
| PIO | CPU 亲自搬数据 |
| DMA | DMA 控制器或设备自己搬数据 |

PIO 中，CPU 需要不断执行类似操作：

```c
DATA = buffer[i];
```

这会占用 CPU。

DMA 中，CPU 只负责告诉 DMA：

```text
数据在内存哪里
要传到哪个设备
传多少
传完后通知我
```

真正的数据搬运由 DMA 完成。

### 3.2 轮询和中断解决的是“怎么知道设备完成”

| 方式 | 谁通知谁 |
|---|---|
| 轮询 polling | CPU 主动反复问设备是否完成 |
| 中断 interrupt | 设备完成后主动通知 CPU |

轮询方式：

```text
CPU：你好了吗？
CPU：你好了吗？
CPU：你好了吗？
```

中断方式：

```text
CPU 先去运行其他进程
设备完成后发中断通知 CPU
```

---

## 4. 中断不一定总比轮询好

中断的优点是：

```text
CPU 不需要一直等待设备
设备完成后主动通知 CPU
```

但是中断本身也有成本：

```text
保存上下文
切换到内核
执行中断处理程序
恢复上下文
可能影响缓存
```

### 4.1 慢速设备适合中断

例如：

```text
键盘
鼠标
磁盘
```

这些设备事件少，等待时间长，用中断更合理。

### 4.2 高速网络设备可能更适合轮询

如果网卡每收到一个包就发一次中断：

```text
包 1 -> 中断
包 2 -> 中断
包 3 -> 中断
...
```

CPU 可能一直在处理中断，没时间处理用户请求。

这可能导致活锁，livelock。

活锁指的是：

```text
系统一直很忙，但没有真正完成有效工作。
```

### 4.3 中断合并 coalescing

中断合并是指设备不为每个事件都发中断，而是等一小段时间，把多个事件合并成一次中断。

例如：

```text
10 个网络包 -> 1 次中断
```

优点：

```text
减少中断次数
降低 CPU 开销
```

缺点：

```text
增加请求延迟
```

---

## 5. DMA 的意义

没有 DMA 时，CPU 既要运行进程，又要搬运数据。

例如进程 1 要写磁盘：

```text
CPU 运行进程 1
CPU 负责把数据拷贝给磁盘
CPU 才能运行进程 2
磁盘执行真正 I/O
```

有 DMA 后：

```text
CPU 运行进程 1
CPU 设置 DMA 请求
DMA 负责搬运数据
CPU 可以运行进程 2
磁盘执行真正 I/O
```

核心区别：

```text
没有 DMA：CPU 是搬运工
有 DMA：CPU 是指挥员，DMA 是搬运工
```

DMA 不一定让磁盘本身更快，但能减少 CPU 被数据搬运占用的时间。

## 6. 操作系统如何访问设备寄存器

操作系统需要访问设备的寄存器，有两种常见方式。

### 6.1 特殊 I/O 指令

在 x86 中，可以用专门的 I/O 指令访问设备端口：

```asm
in
out
```

例如：

```asm
out 0x3F8, al
```

这里的 `0x3F8` 是 I/O 端口号，不是普通内存地址。

这种方式下，设备寄存器位于独立的 I/O 端口空间。

### 6.2 I/O 端口空间在哪里

I/O 端口空间不是 DRAM 内存的一部分。

它是 CPU 架构和硬件总线定义的一套独立编号空间。

```text
内存地址空间：
    地址 -> DRAM 或内存映射设备

I/O 端口空间：
    端口号 -> 设备寄存器
```

### 6.3 内存映射 I/O

另一种方式是 memory-mapped I/O。

它把设备寄存器映射到物理地址空间中的某些地址。

例如：

```c
volatile int *status = (int *)0xFE000000;
```

CPU 看起来像在访问内存，但硬件会把访问转发到设备寄存器，而不是 DRAM。

### 6.4 两种方式对比

| 方式 | 访问对象在哪里 | 访问方式 |
|---|---|---|
| 特殊 I/O 指令 | 独立 I/O 端口空间 | `in/out` |
| 内存映射 I/O | 物理地址空间的一部分 | 普通 load/store |

## 7. 设备驱动程序

每种设备都有自己的接口和协议。

操作系统不可能让文件系统、应用程序都知道各种设备细节，所以需要设备驱动程序。

设备驱动程序的作用是：

```text
把操作系统的通用请求翻译成具体设备能理解的寄存器操作。
```

例如文件系统只想说：

```text
读某个块
写某个块
```

但硬件需要：

```text
写扇区数量寄存器
写 LBA 地址寄存器
写 READ/WRITE 命令
读写数据端口
```

驱动程序负责完成这种翻译。

---

## 8. 文件系统栈

一个简化的 I/O 软件栈可以表示为：

```text
应用程序
    |
POSIX API: open/read/write
    |
文件系统
    |
通用块接口
    |
通用块层
    |
具体块设备接口
    |
设备驱动程序
    |
硬件设备
```

### 8.1 应用程序

应用程序只使用统一接口：

```c
open()
read()
write()
close()
```

它不关心底层是 HDD、SSD、U 盘还是其他设备。

### 8.2 文件系统

文件系统管理：

```text
文件名
目录
inode
文件偏移
权限
缓存
```

### 8.3 通用块层

通用块层管理块设备请求：

```text
读哪个块
写哪个块
请求如何排队
请求是否合并
先处理哪个请求
```

### 8.4 设备驱动程序

驱动程序负责具体设备协议：

```text
寄存器怎么写
命令怎么发
中断怎么处理
DMA 怎么设置
错误码怎么解释
```

## 9. 块 block 和扇区 sector

### 9.1 扇区 sector

扇区是磁盘硬件提供的基本读写单位。

传统大小通常是：

```text
512 字节
```

磁盘可以看成一个扇区数组：

```text
sector[0]
sector[1]
sector[2]
...
sector[n-1]
```

如果磁盘有 `n` 个扇区，那么扇区编号范围是：

```text
0 到 n-1
```

这就是磁盘驱动器的地址空间。

### 9.2 块 block

块通常是操作系统或文件系统使用的单位。

常见大小：

```text
4KB
```

如果：

```text
1 个扇区 = 512B
1 个块 = 4KB
```

那么：

```text
1 个块 = 8 个扇区
```

因为：

```text
4096 / 512 = 8
```

### 9.3 区别

| 概念 | 层次 | 常见大小 |
|---|---|---|
| 扇区 sector | 磁盘硬件单位 | 512B |
| 块 block | 操作系统 / 文件系统单位 | 4KB |

---

## 10. 原子写与不完整写入

磁盘通常只保证单个 512B 扇区写入是原子的。

原子的意思是：

```text
要么完整完成
要么完全不完成
不会只写一半
```

### 10.1 单扇区写入

写一个 512B 扇区时，结果通常是：

```text
完整写入成功
或者写入失败
```

### 10.2 多扇区写入

如果一次写 4KB，即 8 个扇区：

```text
sector 100
sector 101
...
sector 107
```

中途突然断电，可能出现：

```text
sector 100-103 已写入新数据
sector 104-107 仍然是旧数据
```

这种情况叫：

```text
不完整写入 torn write
```

这对文件系统一致性非常重要，因此文件系统需要日志等机制处理崩溃恢复。

## 11. 磁盘访问性能假设

磁盘接口虽然只说可以读写扇区，但操作系统通常还默认一些性能规律：

```text
访问相邻块比访问相隔很远的块更快
顺序访问比随机访问快很多
```

这被称为磁盘驱动器的“不成文合同”。

### 11.1 为什么顺序访问快

对于机械硬盘，随机访问需要：

```text
移动磁头
等待盘片旋转
再读写数据
```

顺序访问时，磁头通常不用频繁移动，盘片顺着转就可以读很多连续数据。

因此：

```text
顺序访问快
随机访问慢
```

## 12. 机械硬盘的物理结构

机械硬盘 HDD 由多个关键部件组成。

### 12.1 盘片 platter

盘片是圆形的硬质表面，上面涂有磁性材料，用磁性变化存储数据。

一个硬盘可能有一个或多个盘片。

### 12.2 表面 surface

每个盘片通常有两个表面：

```text
上表面
下表面
```

每个表面都可以存储数据。

### 12.3 磁头 disk head

每个被使用的数据表面通常对应一个磁头。

因此，如果有 `n` 个盘片，并且每个盘片两面都用于存储数据，则通常有：

```text
2n 个磁头
```

更严谨地说：

```text
磁头数量 = 被使用的数据表面数量
```

### 12.4 主轴 spindle

所有盘片都连接在同一个主轴上，由电机带动旋转。

旋转速度通常用 RPM 表示：

```text
RPM = Rotations Per Minute
```

例如 10000 RPM：

```text
10000 转/分钟
= 10000 / 60 转/秒
≈ 166.7 转/秒
```

转一圈需要：

```text
1 / 166.7 秒 ≈ 6 ms
```

### 12.5 磁道 track

盘面上的同心圆叫磁道。

```text
外圈磁道
中间磁道
内圈磁道
```

数据沿着磁道存储。

### 12.6 磁盘臂 disk arm

磁盘臂带动磁头移动到目标磁道上。

所有磁头通常固定在同一个磁盘臂组件上，因此：

```text
所有磁头通常一起移动
```

它们不能各自独立移动到不同半径位置。

---

## 13. 机械硬盘访问一次数据的基本过程

一次机械硬盘读写通常包括：

```text
1. 寻道 seek
   磁盘臂移动磁头到目标磁道

2. 旋转等待 rotational latency
   等目标扇区转到磁头下面

3. 数据传输 transfer
   磁头读写数据
```

机械硬盘随机访问慢，主要是因为前两步涉及机械运动。

## 14. 磁道偏斜 track skew

### 14.1 问题

顺序读取时，可能会从一个磁道的最后一个扇区读到相邻磁道的第一个扇区。

但是磁头从一个磁道移动到相邻磁道需要时间。

如果相邻磁道的扇区编号完全对齐，磁头移动过去时可能已经错过了目标扇区，需要再等盘片转一整圈。

### 14.2 解决方法

磁道偏斜就是让相邻磁道的扇区编号故意错开。

例如磁道偏斜为 2 块：

```text
下一条磁道的起始扇区比上一条磁道错开 2 个扇区位置
```

这样磁头移动到下一磁道时，目标扇区刚好转到磁头下面。

### 14.3 作用

磁道偏斜的作用是：

```text
提高跨磁道顺序读写性能
避免每次跨磁道都等待完整旋转周期
```

## 15. 多区域磁盘 multi-zoned disk

### 15.1 为什么外圈扇区更多

磁盘外圈周长比内圈更长。

因此外圈磁道可以容纳更多扇区。

如果内圈和外圈扇区数一样，外圈空间会被浪费。

### 15.2 区域 zone

现代磁盘通常把磁道分成多个区域。

同一区域内：

```text
每条磁道扇区数相同
```

不同区域之间：

```text
外圈区域每条磁道扇区更多
内圈区域每条磁道扇区更少
```

例如：

```text
外圈区域：每条磁道 500 个扇区
中间区域：每条磁道 400 个扇区
内圈区域：每条磁道 300 个扇区
```

### 15.3 外圈顺序读写可能更快

磁盘转速固定，外圈一圈包含更多扇区。

所以单位时间内经过磁头的数据更多。

因此：

```text
外圈顺序传输率通常高于内圈
```

## 16. 磁盘缓存 cache / track buffer

现代磁盘内部通常有缓存。

这块缓存是磁盘驱动器内部的一小块内存，例如：

```text
8MB
16MB
32MB
64MB
```

它也可以叫：

```text
disk cache
track buffer
```

### 16.1 读缓存

当磁盘读取某个扇区时，可能顺便把后续连续扇区也读入缓存。

例如操作系统请求：

```text
读取 sector 100
```

磁盘可能顺便缓存：

```text
sector 101
sector 102
sector 103
...
```

如果后面操作系统继续顺序读取，就可以直接从缓存返回数据。

### 16.2 写缓存

写入时，磁盘可以先把数据写到缓存，然后再写入盘片。

这里有两种策略。

---

## 17. write back 和 write through

### 17.1 write back 回写缓存

流程：

```text
数据先写入磁盘缓存
磁盘立即报告写入完成
之后再真正写入盘片
```

优点：

```text
速度快
```

缺点：

```text
如果断电，缓存中的数据可能还没落盘
可能导致数据丢失或文件系统不一致
```

### 17.2 write through 直写缓存

流程：

```text
数据必须真正写入盘片
磁盘才报告写入完成
```

优点：

```text
更安全
```

缺点：

```text
速度较慢
```

---

## 18. 文件系统为什么关心磁盘缓存

文件系统有时要求写入按特定顺序完成。

例如日志文件系统可能要求：

```text
先写日志
再写真正的数据结构
```

如果磁盘使用 write back，可能出现：

```text
磁盘告诉操作系统日志已经写完
但日志其实还在磁盘缓存里
没有真正写入盘片
```

这时如果断电，文件系统一致性可能出问题。

因此文件系统可能需要使用 flush 命令，要求磁盘把缓存中的数据真正写到稳定存储介质上。

## 19. IDE 磁盘驱动程序展示的核心知识

xv6 的 IDE 驱动代码不是让我们记端口号，而是展示真实驱动的基本结构。

一个驱动程序通常要做：

```text
1. 接收上层 I/O 请求
2. 把请求加入设备请求队列
3. 等设备 ready
4. 写设备寄存器，启动 I/O
5. 让发起请求的进程睡眠
6. 设备完成后发中断
7. 中断处理程序处理结果
8. 唤醒等待进程
9. 启动队列中的下一个请求
```

### 19.1 驱动程序是翻译层

上层请求：

```text
读某个磁盘块
写某个磁盘块
```

底层设备操作：

```text
写扇区数量寄存器
写 LBA 地址寄存器
写命令寄存器
从数据端口读写数据
```

驱动程序负责在二者之间转换。

### 19.2 进程等待 I/O 时会睡眠

进程发起磁盘请求后，不会一直占着 CPU 等待。

它会：

```text
sleep
```

等磁盘完成后，中断处理程序会：

```text
wakeup
```

这样 CPU 可以在磁盘工作期间运行其他进程。

### 19.3 驱动程序需要同步

多个进程可能同时提交请求，中断处理程序也可能同时修改请求队列。

所以驱动程序需要锁保护共享数据结构。

## 20. 本部分核心总结

### 20.1 I/O 设备部分

```text
设备通过寄存器与操作系统交互。
PIO 表示 CPU 搬数据。
DMA 表示 DMA 或设备搬数据。
轮询表示 CPU 主动检查设备状态。
中断表示设备完成后通知 CPU。
驱动程序负责隐藏具体硬件细节。
```

### 20.2 磁盘部分

```text
磁盘可以抽象成扇区数组。
文件系统通常按块访问磁盘。
单扇区写通常是原子的。
多扇区写可能出现不完整写入。
顺序访问通常比随机访问快。
机械硬盘慢主要因为寻道和旋转延迟。
磁道偏斜用于优化跨磁道顺序读取。
多区域磁盘用于充分利用外圈空间。
磁盘缓存可以提高性能，但 write back 有一致性风险。
```

---

## 21. 易错点总结

### 易错点 1：PIO 和轮询不是一回事

```text
PIO 讨论谁搬数据。
轮询讨论如何知道设备完成。
```

### 易错点 2：DMA 和中断也不是一回事

```text
DMA 负责搬数据。
中断负责通知 CPU。
```

常见组合是：

```text
DMA + 中断
```

即 DMA 搬完数据后，设备通过中断通知 CPU。

### 易错点 3：I/O 端口空间不是内存

```text
I/O 端口号不是普通内存地址。
```

内存映射 I/O 才是把设备寄存器映射到物理地址空间。

### 易错点 4：块和扇区不是完全一样

```text
扇区是硬件单位。
块是文件系统或操作系统单位。
```

一个块可能包含多个扇区。

### 易错点 5：中断不一定永远优于轮询

高速网络设备中，如果每个包都触发中断，可能导致活锁。

高负载场景下，轮询可能更可控。

### 易错点 6：write back 不等于真正落盘

磁盘报告写入完成，不一定说明数据已经真正写到盘片。

如果使用 write back，数据可能还在缓存中。

### 易错点 7：多个磁头不是独立移动

机械硬盘中多个磁头通常固定在同一个磁盘臂组件上。

它们通常一起移动，而不是各自独立移动。

## 22. 简短记忆版

```text
设备交互：
    状态寄存器看状态
    命令寄存器发命令
    数据寄存器传数据

PIO：
    CPU 搬数据

DMA：
    DMA 搬数据

轮询：
    CPU 主动问设备好了没

中断：
    设备主动通知 CPU 我好了

驱动程序：
    把通用 I/O 请求翻译成具体硬件操作

磁盘：
    看成扇区数组

扇区：
    硬件单位，传统 512B

块：
    文件系统单位，常见 4KB

机械硬盘慢：
    寻道 + 旋转延迟

顺序访问快：
    少移动磁头，少等待旋转

磁道偏斜：
    相邻磁道扇区错开，方便连续读

多区域磁盘：
    外圈磁道放更多扇区

磁盘缓存：
    提高性能，但 write back 有一致性风险
```
