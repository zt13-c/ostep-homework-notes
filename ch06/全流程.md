# 进程的完整生命周期：从创建到消亡（五幕剧）

## 🎬 第一幕：受命于天，克隆降世（fork 诞生）

**当前状态：** Shell 进程（父）在用户态运行。你敲下 `./hw4`，Shell 调用 glibc 的 fork()，触发陷阱陷入内核。

### 附体与划地

CPU 提升至特权级 0。OS 借用 Shell 的内核栈开始执行。OS 在真实的物理内存中，分配了全新的 PCB 和 8KB 内核栈。

### 复印灵魂与地图

- OS 将 Shell 的 PCB 和内核栈数据原样物理拷贝给子进程。
- OS 申请全新的物理页作为子进程的新页表（PGD），抄录父进程的虚拟到物理映射坐标，并双双打上**"只读"标记**（设下 COW 写时复制的局）。
- OS 将这本新页表的绝对物理地址，记录在子进程的 PCB 中。

### 共享上帝视角的挂载

OS 将新 PCB 和新内核栈的物理地址，映射进 3GB-4GB 的内核全局共享页表（例如虚拟地址 `0xC0080000`）。瞬间，全系统所有进程的内核空间都"看"到了这个新栈。

### 暂不登基

子进程被标记为"就绪"，丢进候车室。OS 执行 sysret 返回用户态，Shell 继续霸占 CPU 执行。

---

## 🎬 第二幕：沧海桑田，灵魂夺舍（Context Switch 切换）

**当前状态：** Shell 时间片耗尽或因 wait() 阻塞，时钟中断触发，OS 调度器选中了新生的子进程。

### 封存旧王

OS 将 CPU 的通用寄存器压入 Shell 的内核栈，并将当前的物理 SP 指针存入 Shell 的 PCB。

### 硬件级暗号对接（TSS 埋伏）

OS 提取子进程 PCB 中的内核栈虚拟地址 `0xC0080000`，悄悄写入物理硬件 **TSS（任务状态段）** 的 esp0 字段。这是给 CPU 硬件留下的"救命锦囊"。

### 地图与导航仪大偷换

- **换 CR3**：OS 从子进程 PCB 掏出新页表的物理首地址，极其暴力地覆写 CPU 的 CR3 寄存器。平民区（0-3GB）瞬间变幻为子进程的视角。
- **换 SP**：OS 将 `0xC0080000` 强行塞进 CPU 的 SP 寄存器。

### 退朝复活

执行 sysret 返回用户态。子进程的 CPU 降至特权级 3，SP 弹回用户栈。子进程以为自己是 Shell，但发现 fork() 返回值为 0，使命召唤开始。

---

## 🎬 第三幕：粉碎过去，脱胎换骨（execve 重生）

**当前状态：** 子进程立刻调用 execve("./hw4")，再次陷入内核（此时用的是自己的内核栈 `0xC0080000`）。

### 撕毁旧地图

OS 无情清空子进程页表中 0-3GB 的所有映射，彻底斩断与 Shell 的 COW 物理羁绊。

### 按图索骥找真身

OS 通过当前目录的 Dentry 表找到 "hw4" 的 Inode 编号，进而在物理硬盘上精准定位到 ELF 文件的物理扇区（LBA 块）。

### 描绘新世界

OS 将这些硬盘扇区地址，登记到已清空的 0-3GB 新页表中（**懒加载机制**）。

### 篡改命运的起点

OS 提取 hw4 ELF 文件的 main 函数虚拟地址（如 `0x401000`），将其硬塞进当前内核栈底，作为即将返回的 PC（程序计数器）值。

### 退朝入世

执行 sysret。特权级降为 3，PC 指针精准落入 `0x401000`。啪！`./hw4` 的第一行代码在用户态跑起来了！

---

## 🎬 第四幕 & 第五幕：陷阱触发，天地倒转（System Call I/O）

**当前状态：** 程序运行到 `write(fd, "hello", 5)`，glibc 将系统调用号 1 放入 rax，执行 syscall 硬件指令。

### 硬件级生死极速（纯电路运作，纳秒级）

1. **权限反转**：硬件立刻将 CPU 特权级从 3 拨到 0。
2. **读取锦囊**：硬件疯狂查询 TR 寄存器指向的 TSS 结构，读出第二幕中 OS 埋下的 `esp0 = 0xC0080000`。
3. **避险换栈**：硬件瞬间将 CPU 的 SP 覆写为 `0xC0080000`（脱离危险的用户栈，回到安全的内核栈）。
4. **封存现场**：硬件自动将用户态的 PC、原 SP、状态标志 EFLAGS 死死压入内核栈。
5. **强行转运**：硬件查 IDT 表，强行将 PC 指向 OS 内核的系统调用总控大门。

### 上帝接管（OS 软件运作）

OS 醒来，低头看 rax 为 1，查表分发至 sys_write 子程序。OS 指挥底层物理总线，将 "hello" 拍到显卡显存或写入磁盘。成功后，将字节数 5 存入 rax。

### 功成身退

OS 执行 sysret。硬件接手，从内核栈弹出用户态的 PC 和 SP，权限降回 3。程序如同做了一场梦，继续在用户态向下执行。

---

## 🎬 终幕：化为尘埃，资源归天（exit 死亡）

**当前状态：** main 函数执行到 `return 0`，底层触发 exit() 系统调用，最后一次陷入内核。

### 宣判死亡

OS 收到注销请求，开始极其冷酷的物理大扫除。

### 焚毁平民区

释放 0-3GB 映射指向的所有物理内存页，并彻底销毁这本专属页表本身（归还那几块存页表的物理内存）。

### 抹除上帝区痕迹

OS 从全局 3GB-4GB 共享内核映射中，抠掉 `0xC0080000` 这个坐标。在真实的物理内存中，回收该子进程的 PCB 和内核栈。

### 唤醒造物主

OS 去阻塞队列中，唤醒当年那个因调用 wait() 而陷入沉睡的父进程（Shell）。时钟流转，调度器再次将 CR3 和 SP 交还给 Shell。

### 余音绕梁

Shell 醒来，在黑色的终端屏幕上，平静地打印出一个崭新的命令提示符：$。
